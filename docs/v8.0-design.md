### Phase 1: Security Hardening (보안 강화)

#### 1.1 Unified safeFetch 래퍼 (H1, H2, H4 해결)
- `src/lib/safeFetch.ts` 생성
  ```typescript
  export async function safeFetch(url: string, init?: RequestInit): Promise<Response> {
    const validation = validateExternalUrl(url)
    if (!validation.valid) throw new Error(`Blocked: ${validation.reason}`)
    return fetch(url, init)
  }
  ```
- 적용 대상:
  - `mcpClient.ts` — 모든 fetch 호출 (5곳)

### Phase 2: Performance Critical (성능 핵심)

**목표**: 번들 30% 감소, 스트리밍 80% 리렌더 감소
**CRITICAL 2, HIGH 3 해결**

#### 2.1 xlsx 동적 import (C1 해결)
- `DataAnalysisTool.tsx`에서 xlsx를 `React.lazy` + `import()` 로드
  ```typescript
  const xlsx = await import('xlsx')
  ```
- Vite 설정에서 xlsx를 별도 chunk로 분리 확인
- **예상 효과**: 초기 로드 -143KB gzip (30% 감소)
- **테스트**: lazy load 동작 확인 2 tests

#### 2.2 스트리밍 rAF 배칭 (C2 해결)
- `useChatStreaming.ts` — useRef로 청크 누적, requestAnimationFrame으로 16ms 간격 flush
  ```typescript
  const pendingRef = useRef('')
  const rafRef = useRef<number>()

  const onChunk = (chunk: string) => {
    pendingRef.current += chunk
    if (!rafRef.current) {
      rafRef.current = requestAnimationFrame(() => {
        setMessages(prev => prev.map(m =>
          m.id === placeholderId
            ? { ...m, content: m.content + pendingRef.current }
            : m
        ))
        pendingRef.current = ''
        rafRef.current = undefined
      })
    }
  }
  ```
- **예상 효과**: 리렌더 50/sec → 16/sec (60fps 프레임 동기화)
- **테스트**: rAF mock으로 배칭 동작 5 tests

#### 2.3 Provider 캐싱 (H8 해결)

- **테스트**: config 변경 시에만 재생성 확인 3 tests

#### 2.4 BM25 이중 토큰화 수정 (H9 해결)
